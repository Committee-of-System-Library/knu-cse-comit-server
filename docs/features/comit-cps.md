# Comit CPS — Context · Problem · Solution

> 이 문서는 Comit의 모든 기능 결정, 우선순위 판단, AI 프롬프트의 기준점이다.
> 새로운 기능을 추가하거나 범위를 결정할 때 반드시 이 문서를 먼저 확인한다.
> 미팅마다 새로운 인사이트가 생기면 이 문서를 업데이트한다.

---

## Context — 우리가 공통으로 깔고 있는 전제

**사용자 환경**
- 경북대 컴퓨터학부 재학생 약 1,400명
- 정보 채널 현황:
  - **에타(에브리타임)**: 전교생 익명 커뮤니티 — 컴학 특화 정보 없음, 출처 불명으로 신뢰도 낮음
  - **선배 건너건너**: 인맥에 따라 정보 격차 발생, 누적되지 않고 휘발

**팀 환경**
- 기획·개발진(8명)이 타겟 유저와 동일한 경북대 컴학 구성원
  → 니즈 검증 비용 0에 수렴, "Scratch your own itch" 상태
- 시도위 + 컴학 인맥을 통한 초기 Seeding 채널 확보
- 런칭 시점: **2026년 3월 말** — 신입생이 "모르는 게 너무 많다"를 실감하는 OT 시즌

**기술 환경**
- BE: Spring Boot 4.0.4 + Java 21, Light DDD 패키지 구조
- 인증: KNU CSE SSO (chcse.knu.ac.kr) Google Workspace OAuth → JWT Resource Server
- SSO JWT Claims: `sub`, `student_number`, `name`, `email`, `major`, `user_type`, `role`
- 우리 DB 저장: `sso_sub`, `name`, `phone`, `nickname`, `student_number`, `major_track`, `agreed_at`, `student_number_visible`

## Problem — 우리가 해결하려는 문제

### P1. 정보의 신뢰도 부재 (핵심)
에타의 익명성은 "컴학생이 썼는지" 알 수 없게 만든다.
같은 질문이라도 실제 컴학생의 답변인지 외부인의 답변인지 구분이 불가능하다.
→ **"컴학생임이 인증된 사람이 썼다"는 사실 하나만으로 정보의 무게가 달라진다.**

### P2. 정보의 접근성 불균형
선배 경험(랩실, 인턴, 전공 선택)은 아는 사람에게만 흘러간다.
인맥이 없는 신입생, 편입생, 소수자 그룹은 같은 정보에 접근하지 못한다.
→ 컴학생 전체가 동등하게 접근할 수 있는 채널이 없다.

### P3. 정보의 휘발
카카오톡 단톡, 에타 게시글, 구두 전달 — 모두 사라진다.
같은 질문이 매 학기 반복되고, 매번 처음부터 답변된다.
→ 쌓이지 않아서 검색도 참고도 불가능하다.

## Solution — 우리가 제공하는 솔루션

### Phase 1 · MVP (2026년 3월 말 런칭)
**"인증된 사람이 쓴 Q&A가 검색 가능하게 쌓인다"**

| 기능 | 해결하는 Problem | 판단 기준 |
|---|---|---|
| KNU SSO OAuth 인증 | P1 — 신뢰도 | 컴학생만 가입 가능 |
| Q&A 게시글 / 댓글 | P2, P3 | 질문-답변 채널 + 아카이브 |
| 대댓글 (댓글의 댓글) | P2, P3 | 질문 흐름을 맥락 단위로 이어감 |
| 도움이요 (댓글) | P1 — 신뢰도 | 유용한 답변 상단 정렬 |
| 좋아요 (게시글) | P2 — 접근성 | 공감 많은 질문 Hot Q&A |
| 자유 태그 + 검색 | P3 — 아카이브 | 축적된 정보 탐색 |
| 닉네임 + 학번 선택 공개 | P1 — 신뢰도 | 완전 익명 없음, 컨텍스트 부여 |

Hot Q&A 인기글은 전체 게시판을 대상으로 최근 7일 반응을 집계해 상위 5개를 노출한다. 점수식은 `최근 7일 좋아요 수 * 5 + 최근 7일 댓글 수 * 3 + 최근 7일 unique 방문자 수 * 2`를 사용한다.

**Phase 1 성공 기준 (런칭 후 2주)**
- 가입자 100명 (컴학 1,400명의 7%)
- 주간 게시글 30개 이상
- 게시글당 평균 댓글 1.5개 이상

### Phase 2 · Retention (검증 후 결정)
- 태그 트래픽 임계치 초과 시 별도 게시판 분리
- 커뮤니티 정체성 강화 기능 (호반우 픽셀, 오늘의 한마디 등)

### Phase 3 · 확장 (시장 검증 후 결정)
- 스터디 매칭, 강의 리뷰, 취업 후기 아카이브, 졸업생 네트워크

## 기능 판단 기준 (이 문서의 핵심 활용법)

새로운 기능 아이디어가 나올 때 아래 질문에 답한다.

```
1. 이 기능은 P1·P2·P3 중 어느 문제를 해결하는가?
2. Phase 1 성공 기준(가입자·게시글·댓글)에 기여하는가?
3. 1개월 일정 안에 구현 가능한 최소 형태가 있는가?

→ 셋 중 하나라도 "아니오"면 Phase 2 이후로 이동한다.
```

**판단 예시**

| 기능 아이디어 | 판단 | 근거 |
|---|---|---|
| 대댓글 (댓글의 댓글) | Phase 1 포함 | 질문-답변 맥락 유지에 직접 기여하므로 현재 범위에 포함 |
| 알림 시스템 | Phase 2 이후 | Retention 기능, MVP 범위 초과 |
| 게시글 검색 | Phase 1 포함 | P3(아카이브 활용) 직결 |
| 팔로우 / 프로필 강화 | Phase 3 이후 | SNS 성격, 현재 Problem과 거리 있음 |
