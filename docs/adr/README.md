# adr/

Architecture Decision Records. 기술적으로 중요한 결정을 내렸을 때 작성.

## 작성 시점

- 기술 스택·라이브러리 선택
- 아키텍처 방향 결정
- 팀 내 논쟁이 있었던 설계 선택

## 파일 네이밍

```
{순번}-{제목}.md
예) 001-api-doc-automation.md
```

## 상태값

- `채택` — 현재 적용 중
- `거절` — 검토했으나 채택하지 않음
- `폐기` — 채택했다가 이후 변경됨
