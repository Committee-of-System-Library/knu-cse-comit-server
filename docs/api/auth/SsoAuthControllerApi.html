SSO 로그인 콜백 처리 GET /auth/sso/callback SSO 로그아웃 POST /auth/sso/logout SSO 로그인 시작 GET /auth/sso/login
GET

SSO 로그인 콜백 처리

auth-server가 전달한 custom JWT와 state를 검증한 뒤, SSO 토큰 쿠키를 심고 프론트 성공 URL로 리다이렉트합니다.

GET/auth/sso/callback

쿼리 파라미터

필드명 타입 필수 여부 설명
state String required 로그인 시작 시 발급한 state와 일치해야 합니다.
token String required auth-server가 내려준 하위 서비스용 custom JWT입니다.

에러 응답 필드

필드명 타입 필수 여부 설명
type String required 문제 유형을 식별하는 URI path입니다.
title String required HTTP 상태를 요약한 제목입니다.
status Integer required HTTP 상태 코드입니다.
detail String required 클라이언트에 노출할 에러 설명입니다.
instance String required 에러가 발생한 요청 경로입니다.
errorCode String required 프론트엔드 분기에 사용하는 안정적인 에러 코드입니다.
invalidFields List<ProblemFieldViolation> optional validation 에러에서만 포함됩니다. 필드명과 검증 메시지 목록입니다.
errorTrackingId String optional 서버 내부 오류에서만 포함됩니다. 로그 추적용 식별자입니다.
timestamp OffsetDateTime required 에러 응답 생성 시각입니다.

에러 코드

errorCode HTTP 상태 type detail 발생 조건
INVALID_REQUEST 400 /problems/common/invalid-request 입력값이 올바르지 않습니다. state 값이 누락되거나 기존 state cookie와 일치하지 않을 때
UNAUTHORIZED 401 /problems/common/unauthorized 인증이 필요합니다. token 서명, issuer, audience, 만료 시간이 유효하지 않을 때

응답 예시

{ }

에러 예시

{
  "type" : "/problems/common/invalid-request",
  "title" : "Bad Request",
  "status" : 400,
  "detail" : "입력값이 올바르지 않습니다.",
  "instance" : "/auth/sso/callback",
  "errorCode" : "INVALID_REQUEST",
  "invalidFields" : [ {
    "field" : "fieldName",
    "message" : "유효하지 않은 입력값입니다."
  } ],
  "timestamp" : "2026-03-24T03:10:00Z"
}
